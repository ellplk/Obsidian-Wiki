# 软件代码设计笔记摘要

> 原始文件：[[../../raw/02_编码与技术/技术笔记/软件代码设计.md|软件代码设计.md]]

## 核心内容

本文档系统性地梳理了构建高质量软件的五大核心概念：面向对象、设计原则、设计模式、编程规范、重构。

### 1. 面向对象 (OOP)
- **历史**：Simula (1960s) → Smalltalk (1970s) → C++ (1980s) → Java (1990s)
- **核心特性**：封装、继承、多态、抽象
- **驱动力**：通过对象组织代码，管理复杂性，提高复用性

### 2. 设计原则
- **SOLID**：SRP、OCP、LSP、ISP、DIP
- **其他**：DRY、KISS、YAGNI、LoD（最少知识原则）
- **目标**：高内聚、低耦合

### 3. 设计模式 (GoF 23种)
- **创建型**：单例、工厂、抽象工厂、建造者、原型
- **结构型**：适配器、桥接、组合、装饰器、外观、享元、代理
- **行为型**：责任链、命令、解释器、迭代器、中介者、备忘录、观察者、状态、策略、模板方法、访问者

### 4. 编程规范
- 命名规范、代码格式化、注释规范
- 文件目录结构、语言特性使用

### 5. 重构
- **定义**：在不改变外部行为的前提下改善内部结构
- **关键**：小步快跑、依赖自动化测试作为安全网
- **场景**：添加功能前的预备性重构、修复 Bug、代码评审后、消除代码坏味道
