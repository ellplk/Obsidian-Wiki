# 单元测试笔记摘要

> 原始文件：[[../../raw/02_编码与技术/技术笔记/单元测试.md|单元测试.md]]

## 核心内容

本文档总结了 Python 单元测试的核心框架、流程与技巧。

### 核心框架
1. **pytest**：Python 主流测试框架
   - `@pytest.fixture`：可重用设置代码，支持多作用域（function/class/module/session）
   - 断言直接使用 `assert`，失败信息详细
   - 自动发现 `test_*.py` / `*_test.py`
   - 插件生态丰富：pytest-cov、pytest-xdist

2. **unittest.mock**：模拟库
   - `Mock` / `MagicMock`：创建替身对象
   - `@patch`：装饰器/上下文管理器，临时替换对象
   - `side_effect`：模拟动态行为、异常、迭代返回值

### 测试流程：Arrange-Act-Assert (AAA)
1. **Arrange**：准备输入数据、设置 Mock、获取被测对象
2. **Act**：调用被测代码
3. **Assert**：验证返回值、对象状态、Mock 调用情况

### 测试小技巧
- 函数编写后在注释标注 to-do 测试项
- 测试时间分配建议：7:2:1（单元:集成:功能）
- 优先验证核心功能，再补充边缘条件

### 注意事项
- 测试隔离：通过 mock 确保只测当前单元逻辑
- 明确测试目标：一函数一目标
- Mock 粒度要精确，避免过于宽泛的 patch
- 合理选择 fixture scope，注意共享状态干扰
