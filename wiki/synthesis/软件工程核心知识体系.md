# 软件工程核心知识体系

## 概述

本知识库中的软件工程内容围绕代码质量、可维护性和设计优雅性展开，涵盖五大核心领域。

## 一、面向对象编程 (OOP)

- **历史**：Simula → Smalltalk → C++ → Java
- **四大特性**：封装、继承、多态、抽象
- **价值**：更好地管理复杂性、提高复用性、适应变化

## 二、设计原则

- **SOLID**：SRP、OCP、LSP、ISP、DIP
- **补充**：DRY、KISS、YAGNI、LoD
- **目标**：高内聚、低耦合

## 三、设计模式

GoF 23 种经典模式：
- **创建型**：单例、工厂、建造者等
- **结构型**：适配器、装饰器、代理等
- **行为型**：观察者、策略、模板方法等

## 四、编程规范

- 命名规范、代码格式化、注释规范
- 文件目录结构、语言特性使用
- 价值：提高可读性、一致性、降低维护成本

## 五、重构

- **定义**：不改变外部行为的前提下改善内部结构
- **关键**：小步快跑、测试驱动
- **场景**：预备性重构、修复 Bug、消除代码坏味道

## 六、单元测试

- **框架**：pytest + unittest.mock
- **模式**：Arrange-Act-Assert (AAA)
- **原则**：隔离性、单一目标、可读性
