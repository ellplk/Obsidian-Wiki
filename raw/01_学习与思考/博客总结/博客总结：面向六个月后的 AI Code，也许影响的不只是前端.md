## 核心框架 
### 1. 核心论点 
AI在编程领域发展迅速，在不远的将来（18 - 24个月甚至更短时间）可能取代初级程序员的编程任务，未来AI代码生成体系将形成“规范驱动→知识沉淀→协议贯通→智能执行”的闭环架构，对编程行业产生全方位、深层次的影响，且这种影响目前对前端开发者冲击较大，后续后端开发者可能面临更大冲击。
### 2. 支撑论据 
- 论据1：Agent具有自主决策和持续工作至任务完成的能力，适合复杂、有价值且容错性高的任务，Coding领域与这些特征高度匹配。当前如Cursor、Windsurf等工具在Agent模式下，已展现出强大的编程辅助能力，如自动修正编译报错、分解复杂任务、实现环境感知与自主决策等，说明AI在编程任务中的应用具有可行性和高效性。 
- 论据2：技术的不断突破解决了AI在编程应用中的诸多难题。例如，Cursor的Codebase Indexing技术可分析代码库，使AI生成更符合项目实际的代码；RAG + Function Call技术让模型能够理解企业知识，尽管存在一些问题，但MCP（Model Context Protocol）的出现有望解决这些问题。MCP作为一种标准化协议，具有统一生态、提高开发效率、解决上下文爆炸和支持动态发现等优势，能实现模型与外部系统的高效协作，为AI在编程领域的进一步发展提供了有力支持。 
- 论据3：行业权威人士的观点也支持这一论点。Anthropic首席执行官Dario Amodei明确表示在初级编码工作上，“替代”现象可能在一年半到两年内出现。随着模型不断发展，人们将更深刻地认识到AI的风险和收益，这从侧面反映出AI在编程领域的变革即将到来 。
## 个人思考 
- **MCP 协议相当于 AI 模型的外挂**：统一外部 api 接口调用方式；解决上下文爆炸等问题。
- **未来模型发展趋势**：MCP 提供给模型调用外部应用 api 的能力，减少 ai 幻觉；Function Call 让模型能自主调用 api，模型自身参数量、解决问题能力也在迅速提升。未来模型可能会发展为多模态 agent 模式，即能够识别图片、语音、视频和文字，提取问题描述，自主使用自身能力/调用外部 api解决问题。 
- **程序员发展前景展望**：因 cursor、windsurf 等 ai ide 对程序开发的巨大提升，程序员花在代码开发的时间将极速减少，初级程序员发展见顶。未来程序员可发展领域：
	- 1.转向产品经理，但需要了解产品发展趋势，规划产品发展，另需要和多方进行协调，难度较大。 
	- 2.转向 ai agent 开发，即针对各应用 api 写 prompt，难度较小，但随着 ai 发展可替代性增强。
	- 3.运维、数据、医疗和法律等ai操作具有高风险领域，需要对每一步操作进行校验，ai 暂无法替代。
- **未来可能发展起的职业**：ai 运行规则设计、ai 生成代码、图片、视频内容检查。
