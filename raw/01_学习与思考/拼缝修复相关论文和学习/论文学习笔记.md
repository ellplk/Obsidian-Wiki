# 2、OminiRefiner[[2511.19990v1.pdf]]
## 用 reference 修复一张已生成图的局部细节"这件事上仍然不行
原因：
1、VAE latent 压缩本身就丢细节。
2、后编辑区域同背景存在色差、内容差、纹理差。
## 解决方法
基于Flux.1 
1、空间错位：在模型层引入交叉注意力，让参考图和原图进行语义匹配。
2、局部性拼缝错位：引入相关损失函数，让模型在mask外保持原图一致，mask内进行修复。
3、数据集多样：包含多种错位。
## 模型架构
**双输入条件 diffusion editor**,基座是 FLUX.1-Kontext-dev(单图编辑模型),用 LoRA(rank 128)
关键改动:把原来"单图 + 文本"的接入方式扩成同时吃 to-be-refined 草图 I 和 reference 图 R,两组 token 之间用 bidirectional attention。这样 reference 的高频细节信息可以在 attention 层面直接被 target 区域查询到,而不是通过文本/CLIP embedding 这种损失严重的 bottleneck 传过去。架构上没有改 FLUX 主干,只是把 conditioning 的接入方式扩展了——这一点跟 Omini-Kontext 那一类用 RoPE trick 做多图输入的工作思路相近,但 OmniRefiner 走的是 SFT + RL 两段式,而不是单纯结构改造。
## 训练方法
**第一阶段:SFT(局部性约束的 dual-input 微调)**

- 基座:FLUX.1-Kontext-dev,LoRA rank 128,单卡 H200,学习率 1×10⁻⁴。
- 监督目标:让模型只在 mask 区域 Ω 内修改,Ω 外的像素严格保留(论文里写的是 "enforced up to numerical tolerance")。
- 这一阶段解决的是全局结构、光照、几何一致性。

**第二阶段:GRPO 强化学习(细节精修)**

- 这一阶段是论文最有意思的部分。作者观察到 SFT 之后,文字笔画、序列号、面料纹理这种 micro-texture 还是欠拟合,原因是 diffusion 的平滑性 + 监督信号在高频区域的天然不平衡。
- 用 GRPO(从 LLM RLHF 借过来,DanceGRPO / Flow-GRPO 已经把它移植到 flow matching 上)在 SFT 模型基础上继续优化。
- **Reward 设计是关键**:patch-wise reward = DreamSim 感知奖励 + masked MSE pixel-level 奖励。DreamSim 是一个跟人类感知对齐的相似度度量(基于 DINO/CLIP ensemble + LoRA 微调),负责"看起来像不像";masked MSE 只在 mask 区域里算 pixel 误差,负责"像素级准不准"。两者结合可以选择性地锐化高频区域,而不会扰动背景。
- 训练配置:4 卡 H200,GRPO 16 trajectories,batch size 4,800 个 fine-tuning steps。
- 论文中的 Figure 5 展示了 DreamSim reward 和 masked MSE reward 在训练过程中的爬升曲线,说明两个 reward 都在稳定收敛。
**数据 — 30K 合成 triplet pipeline**

- 30K 三元组:(degraded target, clean reference, ground truth)。
- 由自动 pipeline 合成:用图像编辑模型 + VLM 引导的 cropping 生成,覆盖多类目、多材质、多退化方式。
## 效果
人脸、logo、文字达到sota。
# PixPerfect [[2512.03247v1.pdf]]
## 解决问题
LDM-based 的 inpainting / local editing 在生成出来后,合成区域和背景在像素级总是对不上,具体是三类 artifact 混在一起:**chromatic shift(色偏)、texture / noise pattern mismatch(纹理或噪声 pattern 不一致)、content discontinuity(内容不连续产生的可见 seam)**。
根因还是 VAE latent compression。VAE decoder是全局注意力，mask内的修改也会影响mask外。
## 解决方法
不在 latent 里折腾,**直接在 pixel space 训一个轻量 GAN-based refiner**,后挂在任何 LDM editing pipeline 后面。三个核心 contribution:

1. **Discriminative pixel space** —— 学一个把 RGB 映射到"对边界处色差/纹理差更敏感"的空间,在那个空间里加监督 loss。
2. **Artifact simulation pipeline** —— 因为真实 LDM 输出没有 GT 对(GT 本来就 ambiguous),作者完全用合成数据,在干净图上注入逼真的 LDM artifact 来构造 (x_gen, x_gt) 训练对。
3. **Pixel-space 直接修** —— 跟 latent 解耦,所以 model-agnostic / task-agnostic / plug-and-play。
## 模型架构
**Refiner G**:`x_pred = G(x_gen, m)`,输入是 LDM 输出的 edited image + binary mask,输出 refined image。

- 基座是 **CMGAN**(Zheng 等的之前工作,ECCV 2022),但做了两处改动:把 bottleneck 的 FC 层换成 **global average pooling**,让网络变成 fully convolutional(于是 resolution-agnostic);再做 **channel pruning**,最终 **41M 参数**——非常轻量。
- 推理时是 **single forward pass**,不需要迭代采样。

**Discriminative tone mapping** fθ:R3→R3f_\theta: \mathbb{R}^3 \to \mathbb{R}^3 fθ​:R3→R3:

- 这是论文最 elegant 的设计。用一个**逐通道的多项式回归**参数化:对每个 RGB 通道 cc c,yc=∑dpc,d xcdy_c = \sum_d p_{c,d}\, x_c^d yc​=∑d​pc,d​xcd​,polynomial degree D=5D=5 D=5。
- 关键:fθf_\theta fθ​ 不是固定的—— **每个 sample 都重新拟合**一组 θ\theta θ。回归输入是 xpredx_{pred} xpred​,回归目标是一张"放大色差"的图 yamp=xgt+β(xpred−xgt)y_{amp} = x_{gt} + \beta(x_{pred} - x_{gt}) yamp​=xgt​+β(xpred​−xgt​),β∼U[20,40]\beta \sim U[20, 40] β∼U[20,40]。回归系数用 **Moore-Penrose pseudoinverse** 闭式求,可微,所以可以端到端 backprop。
- mask 内外做 **balanced sampling**(各取等量像素),避免 mask 大小影响。
- 直觉:这个 fθf_\theta fθ​ 学的是"把这张图里前景和背景之间被忽略的 subtle 差异放大"的 tone mapping。在原 RGB 空间里 L1/LPIPS 看不见的 1-2 灰阶色差,投到 fθf_\theta fθ​ 出来的空间里就被放大成肉眼可见的差,于是 loss 就有梯度。

注意:**fθf_\theta fθ​ 只在训练阶段算 loss 用,推理时不需要** 。这点很巧——推理时只有 G,所以也不依赖 xgtx_{gt} xgt​。
## 缺陷
未开源
# HiFi-Inpaint [[2603.02210v2.pdf]]
## 解决问题
**任务定义**:给一张被 mask 掉的人物图(masked human image)和一张干净的产品参考图(product reference),生成"人拿着这个产品"的最终合成图。卖点必须是**产品细节高保真**——logo 字、纹理图案、瓶身形状不能跑偏,因为这是电商广告的硬需求,产品长得不像就不能用。

作者归纳了三个核心痛点:

- **缺数据**:真实"人手持指定产品"的配对数据极难收集(同一个产品 + 同一个人在不同 pose 下手持,工业上需要专门拍摄)
- **现有模型不会聚焦细节**:Paint-by-Example、ACE++、Insert Anything、FLUX-Kontext 这些通用 reference-based editing 方法,把产品大致形状放对了,但 logo 字、纹理这种高频细节经常糊掉或漂走
- **监督粒度太粗**:只在 RGB 空间算 L1/LPIPS,对高频区域不敏感,模型没有动力去学这些细节
## 解决方法
三件套:

1. **HP-Image-40K 数据集**:用 self-synthesis pipeline 自动造 43,632 对训练数据
2. **Shared Enhancement Attention (SEA)**:在 DiT 的 dual-stream block 里开一条额外的高频 token 分支,用同样的权重再 forward 一次,把高频信息额外注入到 mask 区域
3. **Detail-Aware Loss (DAL)**:在高频图上算 loss,而不是只在 RGB 上算
## 模型架构
基座是 **FLUX.1-Dev**(注意不是 FLUX.1-Kontext)。

**输入** 三样:text prompt、masked human image(target,要修复)、product image(reference)。VAE encoder 把两张图都编成 latent,text encoder 出 text token。**额外**对 reference 图做 high-frequency extraction(类似 high-pass filter,留下 logo/边缘),编码成 high-frequency token。

**Token 拼接顺序**(图里从左到右):text token(黄)→ target latent token(浅绿)→ reference latent token(深绿)→ HF token(深绿斜线)→ noise token(灰)。

**Shared Enhancement Attention(关键创新)**

在 dual-stream DiT block 里,作者**复制一条额外的视觉分支**专门处理高频 token,但**这条分支和原分支共享权重**(这就是 "Shared" 的由来——零额外参数)。

数学上:设 ziz_i zi​ 是原分支(text + target + reference + HF + noise)的 token,zi′z_i' zi′​ 是只把 reference token 替换成 HF token 后的 token 序列。同一个 block Bi(⋅)B_i(\cdot) Bi​(⋅) 分别 forward 一次:

- 原分支输出 Bi(zi)B_i(z_i) Bi​(zi​) 是正常 attention 结果
- 高频分支输出 Bi(zi′)B_i(z_i') Bi​(zi′​) 是"只看高频"的 attention 结果
- 二者结合:**原分支输出 + α · Mask(高频分支输出, M_ds)**

其中 α 是**可学习权重**(论文说比固定为 1 效果好),M_ds 是 down-sampled mask,只让高频分支的输出注入到 mask 区域(背景不动)。

**Single-stream blocks** 后半段就是标准 FLUX 那套,所有 token 拼一条做 full attention,不再分流。

直觉:模型对同一个 patch 算两遍 attention——一遍正常看完整 reference,一遍只看 reference 的高频骨架。两次 forward 的差异作为细节增强信号,**只作用在 mask 区域**(就是要修复的人物身上要"贴"产品的地方),保证背景不被打扰。妙处在于**零额外参数**——共享权重,只多一次 forward。
## 训练方法
**HP-Image-40K 数据构造**(图中左栏)

四步 pipeline,目的是自动造 (reference, target with mask, GT) 三元组:

- **Step 1: Diptych Synthesis** —— 用 prompt `"A diptych. Left: <product>. Right: <human holding product>"` 喂给 **FLUX**,一次性生成左右拼图。这是巧妙的地方:**FLUX 单次去噪保证左右两个产品在视觉上高度一致**(同 latent、同采样轨迹),如果两次独立 generation 根本对不上。
- **Step 2: Diptych Segmentation** —— edge detection 把左右切开,左图 = 产品 reference,右图 = 人持产品(target/GT)。
- **Step 3: Semantic Filtering** —— YOLOv8 在右图里检测产品 → 算 detected box 和左图的 **CLIP similarity** → 低相似度扔掉。这一步把 FLUX 偶尔生成的"左右产品不一致"的样本过滤掉。
- **Step 4: Textual Filtering** —— **Intern-VL** 分别 OCR 左图和右图上的产品文字(比如 "ZENLUX"),要求两边识别出的文字 overlap。专门保证 logo/品牌字一致——这是这个工作 selling point 的命门,如果训练数据里左右文字本来就不一致,模型学了也白学。

**Detail-Aware Loss (DAL)**

预测出来的 latent 经 VAE decode → 得到预测图 → 提取高频图 → 跟 GT 的高频图算 loss(L1/MSE)。这一项**直接在像素空间高频图上监督**,绕过 LDM 标准的 latent-space flow matching loss 对高频不敏感的问题。

**总 loss** = 标准 flow matching loss + DAL。两个监督信号:一个在 latent 上(粗),一个在像素高频图上(细)。

**训练配置**

- 基座:FLUX.1-Dev
- 数据:HP-Image-40K(40K 自合成)+ 14K 内部真实数据
- LR 5e-5,batch size 24,10000 steps
- 分辨率 1024×576(竖版,适合人物全身镜头)
- HP-Image-40K 切 1000 张做 test set,训练时不见
